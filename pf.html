Programming Fundamentals

CH 1. An Overview of Computers and Programming Languages

Introduction:

Software and programming languages are crucial for modern computing. The internet and computers are widely used, even by young students. C++ is a programming language for developing software. Before starting to program, it's helpful to understand basic computer concepts and history.

A Brief Overview of the History of Computers:

  • The first device to carry out calculations is abacus. It uses a system of sliding beads in a rack for addition and subtraction. (only for addition and subtraction)
  • Pascaline, developed in 1642 by Blaise Pascal, was a calculating device. It had eight movable dials on wheels and could calculate sums up to eight figures long. (only for addition and subtraction)
  • Later, Leibniz invented the first device that could handle multiplication and division operations
  • In the early and mid-1800s, Charles Babbage, an English mathematician and physical scientist, designed two calculating machines: the difference engine and the analytical engine. The difference engine could perform complex operations such as squaring numbers automatically. Babbage built a prototype of the difference engine, but did not build the actual device. The first complete difference engine was completed in London in 2002, 153 years after it was designed. It consists of 8,000 parts, weighs five tons, and measures 11 feet long.
  • The first computer-like machine was the Mark I. It was built, in 1944, jointly by IBM and Harvard University under the leadership of Howard Aiken. Punched cards were used to feed data into the machine. The Mark I was 52 feet long, weighed 50 tons, and had 750,000 parts.
  • 1946, the Electronic Numerical Integrator and Calculator (ENIAC) was built at the University of Pennsylvania. It contained 18,000 vacuum tubes and weighed some 30 tons.
  • The computers that we know today use the design rules given by John von Neumann in the late 1940s. His design included components such as an arithmetic logic unit, a control unit, memory, and input/output devices.
  • In 1951, the Universal Automatic Computer (UNIVAC) was built and sold to the U.S. Census Bureau.
  • In 1956, the invention of transistors resulted in smaller, faster, more reliable, and more energy-efficient computers. This era also saw the emergence of the software development industry, with the introduction of FORTRAN and COBOL, two early programming languages.
  • In 1977, Stephen Wozniak and Steven Jobs designed and built the first Apple computer in their garage.
  • In 1981, IBM introduced its personal computer (PC). In the 1980s, clones of the IBM PC made the personal computer even more affordable.

Elements of Computer System:

  1. Hardware:

    2. Hardware consists of CPU, RAM, I/O devices and secondary storage.

    • Input devices are keyboard, mouse, and secondary storage.
    • Output devices include screens and printers.

  2. CPU

    • CPU is the brain of the computer.
    • It carries out arithmetic and logical operations.

  3. RAM

    • RAM is directly connected to the CPU. Programs must first be loaded inside the RAM in order for them to be sent to the CPU.
    • Main memory is an ordered sequence of cells, called memory cells.
    • Each cell has a location in the main memory called the address of the cell.
    • When the computer turns off, all data in RAM is lost.

  4. SECONDARY STORAGE:

    • The need for secondary storage can be explained from the fact that all data in the main memory is lost each time a computer shuts down.
    • Secondary storage is directly connected with RAM.
    • Examples include Hard drives, Solid state drives, flash drives, CD/DVD etc.

  5. I/O devices:

    • The devices that feed data and programs into computers are called input devices.
    • The keyboard, mouse, scanner, camera, and secondary storage are examples of input devices.
    • The devices that the computer uses to display results are called output devices.
    • A monitor, printer, and secondary storage are examples of output devices.

  6. Software

    • System programs control the computer. The system program that loads first when you turn on your computer, is called the operating system. Without an operating system, the computer is useless.
    • The operating system handles the overall activity of the computer and provides services.
    • Some of these services include memory management, input/output activities, and storage management.
    • The operating system has a special program that organizes secondary storage so that you can conveniently access information.
    • Some well-known operating systems are Windows, Mac OS X, Linux, and Android.
    • Application programs perform a specific task.
    • Word processors, spreadsheets, and games are examples of application programs.
    • The operating system is the program that runs application programs.

Processing a C++ Program

The following steps, as shown in Figure 1-2, are necessary to process a C++ program.

  1. Create Source Code

    • Write your C++ program using a text editor.
    • Save the file with a .cpp extension.
    • Ensure the code adheres to C++ syntax and semantics.

  2. Preprocessing

    3. The preprocessor handles directives that begin with #.
    Common directives include:

    • #include: Includes header files containing declarations of functions, variables, and classes.
    • #define: Defines macros for code substitution.
    • #ifdef, #ifndef, #else, #endif: Conditional compilation.
      • The preprocessor expands macros and includes header files, creating a modified source code.

  3. Compilation

    • The compiler translates the preprocessed source code into machine-readable object code.
    • It checks for syntax errors, semantic errors, and type mismatches.
    • The compiler generates object files with a .o or .obj extension.

  4. Linking

    • The linker combines multiple object files and libraries into a single executable program.
    • It resolves references to external functions and variables.
    • Libraries provide pre-written code for common tasks.
    • The linker produces the final executable file, typically with a .exe extension on Windows or no extension on Linux/macOS.

  5. Loading

    • The operating system loads the executable file into memory for execution.
    • The loader sets up the program's environment and initializes necessary data structures.

  6. Execution

    • lThe CPU executes the instructions in the loaded program.
    • The program interacts with the operating system, hardware, and other programs as needed.
    • The program continues to run until it terminates normally or encounters an error.
