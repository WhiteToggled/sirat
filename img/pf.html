Programming Fundamentals

Chapter 1: An Overview of Computers and Programming Languages

Introduction:

In the past, the internet was not as common as it is today. Now, students use the internet to help them with their schoolwork and many other things. Computers need software to work, and software is created using programming languages. C++ is a programming language that can be used to create software.

A Brief Overview of the History of Computers

The abacus, an ancient device using beads for calculations, was the first calculating machine. In the 17th century, Blaise Pascal invented the Pascaline, which could add and subtract. Later, Gottfried von Leibniz created a machine that could perform all four basic arithmetic operations.

In the 1800s, Charles Babbage designed the difference engine and the analytical engine, which could perform complex calculations. Ada Lovelace, considered the first computer programmer, wrote programs about Babbage's machines.

The idea of storing information on punched cards, used in the weaving industry, proved to be important for the development of computers.

At the end of the 19th century, Herman Hollerith invented a calculating machine that used punched cards to store data. Hollerith's machine was immensely successful, leading to the creation of IBM.

The first computer-like machine was the Mark I, built-in 1944. It was followed by the ENIAC in 1946, which was much faster and more powerful. This machine also used punched cards to feed data into the machine.

John von Neumann's design rules for computers, which included components such as an arithmetic logic unit, a control unit, memory, and input/output devices, are still used today.

The invention of transistors in 1956 led to smaller, faster, more reliable, and more energy-efficient computers. This led to the development of early programming languages like FORTRAN and COBOL. Later, integrated circuit "chips" replaced transistors, vastly increasing processing power.

The invention of the microprocessor in 1970 enabled the development of personal computers, with milestones including the first Apple computer in 1977 and IBM’s PC in 1981. By the mid-1990s, affordable PCs became widely accessible.

Modern computers are now powerful and versatile, supporting applications like artificial intelligence, expert systems, and GPS. They facilitate tasks from checking email to making reservations and managing finances. Despite different types (mainframe, midsize, micro), all computers share some basic elements.

Elements of a Computer System

A computer is an electronic device that can perform commands. The basic commands that a computer performs are input (get data), output (display result), storage, and performance of arithmetic and logical operations. There are two main components of a computer system: hardware and software.

Hardware

Major hardware components include the central processing unit (CPU), main memory (RAM), input/output devices, and secondary storage. Examples of input devices are the keyboard, mouse, and secondary storage. Examples of output devices are the screen, printer, and secondary storage.

Central Processing Unit and Main Memory

The CPU is the "brain" of a computer and the most expensive piece of hardware in a computer. The more powerful the CPU, the faster the computer. Arithmetic and logical operations are carried out inside the CPU. Some components are shown in the given figure. RAM is connected directly to the CPU. All programs and data must be loaded into main memory before they can be used. When the computer is turned off, everything in the main memory is lost. Main memory is an ordered sequence of cells called memory cells, and each cell has a unique location called the address of the cell. These addresses help access the information stored in the cell. Today's computers come with main memory consisting of millions to billions of cells. The content of a cell can be either a programming instruction or data and is stored as sequences of 0s and 1s. The memory addresses are also expressed as sequences of 0s and 1s.

Secondary storage

Programs and data must be loaded into main memory for processing, but since main memory is cleared when the computer is turned off, information needs to be saved on a separate device for permanent storage. This permanent storage, called secondary storage, includes devices like hard disks, flash drives, and CD-ROMs, which are directly connected to main memory to allow data transfer.

Input /Output Devices

Hardware

To perform tasks, a computer needs input devices to receive data and programs and output devices to display results. Examples of input devices include the keyboard, mouse, scanner, camera, and secondary storage. Output devices include the monitor, printer, and secondary storage.

Software

Software consists of programs designed for specific tasks, written in programming languages, and categorized into system and application programs. 

System programs manage the computer, with the operating system (OS) loading first to handle activities like memory, storage, and input/output management. Popular OS examples include Windows 10, Mac OS X, Linux, and Android.

Application programs perform specific tasks, such as word processing, spreadsheets, and games, running under the control of the OS.

The Language of a Computer

When you press a key, like "A," the computer stores it as a digital signal - a sequence of 0s and 1s, or binary code inside the main memory.

Computers, as electronic devices, use digital signals (0s for low voltage, 1s for high voltage) rather than analog signals, which are continuous waveforms.

Digital signals allow precise data transfer, unlike analog, which degrades with copying. This binary system is called machine language.

Each 0 or 1 is a bit, and a sequence of 8 bits forms a byte. Table 1-1 summarizes the terms used to describe various numbers of bytes.

Each character (letter, number, or symbol) on your keyboard is encoded as a unique sequence of bits. The most common encoding scheme is the American Standard Code for Information Interchange (ASCII), which is a 7-bit system with 128 characters numbered 0 to 127.

For example, in ASCII, "A" is represented by the binary code 1000001 (position 65) and is the 66th character. This is because, in the ASCII data set, the position of the first character is 0. Each ASCII code corresponds to a binary representation of the character's position in the set.

Inside the computer, each character is stored as a byte (8 bits). ASCII, a 7-bit code, requires an extra 0 added to the left for an 8-bit representation, so "A" (ASCII 65) becomes 01000001, and "3" (ASCII 51) becomes 00110011. 

Unicode is a larger encoding scheme that uses 16 bits (2 bytes) to represent 65,536 characters, supporting a wide range of global languages beyond English. It was created to represent a variety of characters and is continuously expanding.

The Evolution of Programming Languages

Machine language is the most basic language of a computer, using binary codes (bits) to provide instructions. Since each computer’s machine language can differ, early programming was difficult and error-prone as it required memorizing specific binary codes and data locations.

To simplify programming, assembly language was developed, using mnemonics instead of binary codes. For the same calculation, assembly language instructions are easier to read:


LOAD rate

MULT hours

STOR wages

An assembler then translates assembly language into machine language, since a computer cannot execute assembly language instructions directly.

High-level languages like BASIC, FORTRAN, COBOL, C++, Java, and Python were developed to make programming even more intuitive.

This code is closer to natural language and easier to understand. To execute high-level code, a compiler is needed to translate the code into machine code.

Processing a C++ Program

To run a C++ program, the code must be translated into machine language, as computers only understand this basic language. Since computers only understand machine language, C++ code must be translated into machine language before execution. This section reviews the steps required to execute C++ programs.

  1. Create Source Code: Use a text editor to write a C++ program, saving it with a cpp extension (e.g., FirstCPPProgram.cpp).

  2. Preprocessor Directives: Any statements starting with #, like #include , called preprocessor directives are processed by the preprocessor.

  3. Compile the Program: The compiler checks the source code for syntax errors and translates it into machine language, producing an object program.

  4. Linking: Use an Integrated Development Environment (IDE) that includes prewritten code from libraries. The linker combines the object program with library code to create executable code.

  5. Loading: The loader loads the executable program into main memory for execution.

  6. Execution: Finally, execute the program.

Analysis–Coding–Execution Cycle

  1. Analyze the problem: Outline the problem requirements and design an algorithm to solve it.
  2. Implement the algorithm: Code it in a programming language like C++ and verify its functionality.
  3. Maintain the program: Use and update the program as needed if the problem scope changes.

Process of algorithm design

  1. Understand the problem fully.
  2. Determine the requirements: Identify if the program needs user interaction, data manipulation, specific output formats, etc. For data manipulation, review sample data to understand its structure; for output, clarify the format and structure of results.
  3. Break the problem into subproblemsif it is complex. Analyze each subproblem individually, repeating the initial analysis and design steps as needed.

Programming Methodologies

Structured Programming

Structured design involves dividing a problem into smaller subproblems, analyzing each subproblem, and obtaining solutions to solve them. These solutions are then combined to solve the overall problem.

This process of implementing structured design is known as structured programming, which is also referred to as top-down design, bottom-up design, stepwise refinement, and modular programming.

Object-Oriented Programming

Object-Oriented Design (OOD) is a programming methodology that begins by identifying components called objects which form the basis of the solution and interact with one another. Each object consists of data and operations that manipulate that data, forming a single unit. The final program in OOD is a collection of interacting objects, implemented through an Object-Oriented Programming (OOP) language. To design and use objects effectively, it’s essential to understand data representation, manipulation techniques, and operational implementation. Operations are implemented using algorithms and separated into functions, as complex programs may involve many operations on a single data element. Some algorithms require decision-making (selection), others involve repeated statements (repetition), and some may require both. C++ uses classes to combine data and operations into a unit.
